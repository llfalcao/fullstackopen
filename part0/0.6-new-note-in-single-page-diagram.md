# 0.6: New Note in Single Page App Diagram

```mermaid
sequenceDiagram
    participant browser
    participant server

    Note right of browser: The browser prevents the default form behavior, which would reload the page on submit.
    Note right of browser: It then creates the new note, redraws the notes, and finally sends the updated notes to the server.
    
    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note_spa
    activate server
    Note left of server: The server adds the new note to the database, then responds with status 201 created.
    deactivate server

    Note right of browser: No further requests are made to the server.
```
